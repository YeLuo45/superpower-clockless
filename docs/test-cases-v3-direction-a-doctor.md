# Test Cases V3 Direction A - Doctor Command

## TC-DOC-001 Successful Host Validation
- Precondition: temp HOME contains a hermes install generated by `install_agent`.
- Steps: run `run_doctor('hermes')` with mocked healthy API.
- Expected: report ok is true and includes passing catalog/config/mcp/skill/api checks.

## TC-DOC-002 Missing Config Detection
- Precondition: temp HOME has no host config file.
- Steps: run `run_doctor('codex')` with mocked healthy API.
- Expected: report ok is false and config/mcp/skill checks explain missing setup.

## TC-DOC-003 API Failure Is Non-Crashing
- Precondition: local files are installed.
- Steps: run doctor with an API URL that raises connection failure.
- Expected: local checks pass, api check fails, command exits 1.

## TC-DOC-004 All-Agent Aggregation
- Precondition: temp HOME installs all supported agents.
- Steps: run `run_doctor('all')`.
- Expected: one report per supported agent, all ok with mocked healthy API.

## TC-DOC-005 JSON CLI Output
- Precondition: temp HOME has a cursor install.
- Steps: call `run(['doctor', '--agent', 'cursor', '--json'])`.
- Expected: stdout is valid JSON with `ok`, `reports`, `agent`, and check names.

## TC-DOC-006 Non-Mutating Behavior
- Precondition: capture mtimes/content of config files.
- Steps: run doctor repeatedly.
- Expected: content remains unchanged.
